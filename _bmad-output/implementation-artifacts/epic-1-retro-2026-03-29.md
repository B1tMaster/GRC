## What Went Well

1. **100% story completion** — All 16 stories delivered across both epics in a single sprint
2. **Full 7-step AI pipeline working end-to-end** — Ingest → Extract Objectives → Extract Risk Appetite → Derive Controls → Map Frameworks → Gap Analysis → Policy Drafting
3. **Deployment resilience** — 8 significant blockers resolved without stalling:
   - Railway CLI auth → pivoted to GraphQL API
   - Docker CMD exec form fix
   - Stale monorepo lockfile
   - Next.js critical security upgrade (15.0.3 → 15.5.14)
   - DeepSeek max_tokens limit correction
   - Polymorphic relationship ID casting
   - CORS configuration for Netlify
   - Database migration via TCP proxy
4. **Architecture pragmatism** — Payload job queue over separate message broker, Zod schemas for LLM validation, monolith-first approach
5. **LLM provider flexibility** — Swapped from OpenAI to DeepSeek + Qwen without major refactoring
6. **Compelling demo output** — Real governance objectives, framework citations, and policy drafts from real documents

---

## What Didn't Go Well

1. **Testing gap** — Structural validation only; no automated E2E tests, no edge case coverage, no regression protection
2. **Silent LLM failures** — Pipeline reported success with 0 results when DeepSeek max_tokens was exceeded; no observability to detect this
3. **Type system friction** — Payload CMS polymorphic relationships required Number() casting across 4+ files; extensive `as any` usage throughout
4. **Infrastructure assumptions broken** — Docker CMD format, .env handling, monorepo lockfile location all failed on Railway
5. **Document truncation** — 60K character limit means large documents (372-page HSBC report) only get ~5% analysed
6. **Langfuse not operational** — Configured but not connected; LLM debugging was done blind through deployment logs

---

## Key Insights

1. Pre-populating `parsedText` and bypassing Zerox is more reliable for demo than the full upload-parse flow
2. LLM API quirks (max_tokens limits, response format differences) are a major source of silent failures — need better observability
3. Railway GraphQL API is more reliable than CLI for programmatic infrastructure management
4. Payload CMS is powerful but has a steep type system learning curve for polymorphic relationships
5. For demo purposes, the first 20 pages of an annual report contain sufficient strategic governance content
6. The provider abstraction layer (DeepSeek/Qwen/OpenAI) will pay dividends as models improve and costs shift

---

## Demo Readiness Assessment

**Target document:** HSBC Annual Report and Accounts 2025 (372 pages, 9.6MB)

| Concern | Risk | Mitigation |
|---------|------|------------|
| Zerox parsing 372 pages | High — 30-60 min, may timeout | Pre-populate parsedText, skip Zerox |
| Truncation to 60K chars (5% of doc) | Medium — misses Risk Review + Governance | First 20 pages have rich strategic content |
| LLM quality on HSBC content | Low — well-structured corporate document | DeepSeek handles financial/governance well |

**Verdict:** Demo-ready with pre-populated text from pages 1-20. Rehearsal run required before any stakeholder presentation.
